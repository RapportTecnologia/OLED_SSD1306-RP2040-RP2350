◆ CTRL

__IOM uint32_t WATCHDOG_Type::CTRL

< WATCHDOG Structure

Watchdog control The rst_wdsel register determines which subsystems are reset when the watchdog is triggered. The watchdog can be triggered in software.

◆ LOAD

__IOM uint32_t WATCHDOG_Type::LOAD

Load the watchdog timer.

The maximum setting is 0xffffff which corresponds to 0xffffff / 2 ticks before triggering a watchdog reset (see errata RP2040-E1).

The maximum setting is 0xffffff which corresponds to approximately 16 seconds.

◆ REASON

__IOM uint32_t WATCHDOG_Type::REASON

Logs the reason for the last reset.

Both bits are zero for the case of a hardware reset.

Both bits are zero for the case of a hardware reset. Additionally, as of RP2350, a debugger warm reset of either core (SYSRESETREQ or hartreset) will also clear the watchdog reason register, so that software loaded under the debugger following a watchdog timeout will not continue to see the timeout condition.
