117// which are known not to contain the right sort of binary, OR as a way to prevent
118// auto-architecture-switch. NOTE: the first partition that can be booted, will be,
119// so if you have a RISC-V binary in the first partition, and auto-arhcitecture-switch enabled, then
120// even if booting under ARM, with an ARM binary in a later partition, the RISC-V binary
121// will be booted by default; setting PICOBIN_PARTITION_FLAGS_IGNORED_DURING_ARM_BOOT_BITS
122// on the partition, will have the RISC-V binary containing partition ignored under ARM
